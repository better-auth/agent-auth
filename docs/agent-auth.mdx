---
title: Agent Auth
description: Give AI agents their own identity and permissions.
---

The Agent Auth plugin gives AI agent runtimes their own identity so they can act **for** users without impersonating them. Instead of sharing user tokens, each agent gets its own keypair, scopes, and role.

**Core principle:** Agents are tools we hire, not users we pretend to be.

## Features

- [Device authorization connect flow](#connecting-an-agent) — user approves in browser, no manual tokens
- Ed25519 keypair-based authentication (agent owns its private key)
- [Scoped permissions](#roles-and-scopes) via roles and scopes
- [Agent session resolution](#agent-session-resolution) via a `before` hook
- [Key rotation](#rotate-key) without downtime
- [Revocation](#revoke-an-agent) with credential wipe
- [Workgroups](#workgroups) — group agents within an organization
- [Agent SDK](#agent-sdk) for agent runtimes
- [MCP server](#mcp-server) for Cursor, Claude Desktop, and other AI tools
- [Pluggable storage](#storage-options) — memory (default), file, or database
- [AAP-compatible JWT claims](#aap-format) (IETF Agent Authorization Profile)

## Installation

```bash
npm install @better-auth/agent-auth
```

### Server Setup

```ts title="auth.ts"
import { betterAuth } from "better-auth"
import { agentAuth } from "@better-auth/agent-auth"

export const auth = betterAuth({
    plugins: [
        agentAuth({
            roles: {
                reader: ["reports.read"],
                writer: ["reports.read", "reports.write", "email.send"],
            },
            defaultRole: "reader",
        })
    ]
})
```

Run the migration to add the necessary tables:

```bash
npx @better-auth/cli migrate
```

### Client Setup

```ts title="auth-client.ts"
import { createAuthClient } from "better-auth/client"
import { agentAuthClient } from "@better-auth/agent-auth/client"

export const authClient = createAuthClient({
    plugins: [
        agentAuthClient()
    ]
})
```

## How It Works

```
1. Agent generates an Ed25519 keypair (private key never leaves the agent)
2. Agent connects via device authorization — user approves in their browser
3. Agent's public key is registered with the app
4. For each request, the agent signs a short-lived JWT with its private key
5. Better Auth verifies the JWT, resolves the agent session, and attaches it to the request
```

The agent's identity is its keypair. The private key stays on the agent's machine. The server only ever sees the public key.

## Connecting an Agent

The recommended way to connect an agent is via the **device authorization flow** (like `gh auth login`). The agent displays a code, the user approves in their browser, and the agent is registered.

> This requires the `deviceAuthorization()` plugin to be enabled alongside `agentAuth()`.

```ts title="auth.ts"
import { betterAuth } from "better-auth"
import { agentAuth } from "@better-auth/agent-auth"
import { deviceAuthorization } from "better-auth/plugins"

export const auth = betterAuth({
    plugins: [
        agentAuth({ /* ... */ }),
        deviceAuthorization(),
    ]
})
```

### Via SDK (Programmatic)

```ts title="agent-setup.ts"
import { connectAgent } from "@better-auth/agent-auth/agent-client"

const result = await connectAgent({
    appURL: "https://myapp.com",
    name: "Finance Agent",
    scopes: ["reports.read", "email.send"],
    openBrowser: true,
    onUserCode: ({ userCode, verificationUri }) => {
        console.log(`Go to ${verificationUri} and enter: ${userCode}`)
    },
})

// result.agentId — the registered agent ID
// result.privateKey — store this securely
// result.publicKey — already registered with the app
```

## Usage

### Create an Agent

A logged-in user registers an agent by providing its public key and desired scopes.

```ts title="dashboard.ts"
const { data } = await authClient.agent.create({
    name: "Finance Agent",
    publicKey: agentPublicKey, // JWK from the agent
    scopes: ["reports.read", "email.send"],
}, { headers })
// data.agentId — store this on the agent side
```

### List Agents

```ts title="dashboard.ts"
const { data: agents } = await authClient.agent.list({}, { headers })
// agents: [{ id, name, status, scopes, role, lastUsedAt, ... }]
```

### Get Agent

```ts title="dashboard.ts"
const { data: agent } = await authClient.agent.get(
    { query: { agentId: "agent-id" } },
    { headers }
)
```

### Update an Agent

```ts title="dashboard.ts"
await authClient.agent.update({
    agentId: "agent-id",
    name: "Renamed Agent",
    scopes: ["reports.read", "calendar.write"],
}, { headers })
```

### Revoke an Agent

Revocation sets the agent's status to `revoked` and wipes its public key from the database.

```ts title="dashboard.ts"
await authClient.agent.revoke({ agentId: "agent-id" }, { headers })
```

### Rotate Key

Accept a new public key for an agent. The old key stops working immediately.

```ts title="dashboard.ts"
await authClient.agent.rotateKey({
    agentId: "agent-id",
    publicKey: newPublicKey,
}, { headers })
```

## Agent Session Resolution

The plugin installs a `before` hook that intercepts every request. When a `Bearer` token containing a JWT is detected:

1. Decodes the JWT to extract the agent ID (`sub` claim)
2. Looks up the agent in the database
3. Verifies the JWT signature against the stored public key
4. Loads the user who owns the agent
5. Attaches an `agentSession` to the request context

### Protecting API Routes

```ts title="api-route.ts"
export async function GET(req: Request) {
    const session = await auth.api.getSession({ headers: req.headers })
    const agentSession = await auth.api.getAgentSession({ headers: req.headers })

    if (session) {
        const reports = await getReportsForUser(session.user.id)
        return Response.json(reports)
    }

    if (agentSession) {
        if (!agentSession.agent.scopes.includes("reports.read")) {
            return new Response("Forbidden", { status: 403 })
        }
        const reports = await getReportsForUser(agentSession.user.id)
        return Response.json(reports)
    }

    return new Response("Unauthorized", { status: 401 })
}
```

The `agentSession` object:

```ts
{
    agent: {
        id: string
        name: string
        scopes: string[]
        role: string | null
        orgId: string | null
        workgroupId: string | null
        createdAt: Date
        metadata: Record<string, unknown> | null
    },
    user: {
        id: string
        name: string
        email: string
    }
}
```

## Agent SDK

The Agent SDK is for **agent runtimes** — the process that makes API calls.

```ts title="agent-runtime.ts"
import {
    createAgentClient,
    generateKeypair,
} from "@better-auth/agent-auth/agent-client"

const { publicKey, privateKey } = await generateKeypair()
// Register publicKey with the app, get back agentId

const agent = createAgentClient({
    baseURL: "https://your-app.com",
    agentId: "agt_abc123",
    privateKey,
})

const res = await agent.fetch("/api/reports/Q4")
const session = await agent.getSession()
```

## MCP Server

The plugin includes MCP server tools for AI tools (Cursor, Claude Desktop, etc.):

| Tool | Description |
|------|-------------|
| `connect_agent` | Connect to an app via device auth |
| `list_agents` | List agent connections |
| `agent_status` | Check connection health |
| `agent_request` | Make an authenticated API request |
| `disconnect_agent` | Revoke and disconnect |

```json title=".cursor/mcp.json"

        }
    }
}
```

## Workgroups

Workgroups group agents by team, project, or department.

```ts
// Create
await fetch("/api/auth/agent/workgroup/create", {
    method: "POST",
    headers,
    body: JSON.stringify({
        name: "Finance Team",
        orgId: "org_123",
        description: "Agents for the finance department",
    }),
})

// List
await fetch("/api/auth/agent/workgroup/list?orgId=org_123", { headers })

// Update
await fetch("/api/auth/agent/workgroup/update", {
    method: "POST",
    headers,
    body: JSON.stringify({ workgroupId: "wg_456", name: "Finance & Accounting" }),
})

// Delete
await fetch("/api/auth/agent/workgroup/delete", {
    method: "POST",
    headers,
    body: JSON.stringify({ workgroupId: "wg_456" }),
})
```

## Roles and Scopes

```ts title="auth.ts"
agentAuth({
    roles: {
        reader: ["reports.read"],
        writer: ["reports.read", "reports.write", "email.send"],
        finance: ["reports.read", "invoices.read", "invoices.write"],
    },
    defaultRole: "reader",
})
```

When creating an agent with a `role`, scopes are automatically resolved from the role config.

## AAP Format

Supports [IETF Agent Authorization Profile](https://datatracker.ietf.org/doc/draft-ietf-oauth-agent-authorization/) compatible JWT claims:

```ts
agentAuth({
    jwtFormat: "aap",
})
```

## Plugin Options

| Option | Type | Default | Description |
|---|---|---|---|
| `roles` | `Record<string, string[]>` | `undefined` | Named roles mapping to scope arrays |
| `defaultRole` | `string` | `undefined` | Default role assigned when none is specified |
| `allowedKeyAlgorithms` | `string[]` | `["Ed25519"]` | Allowed key algorithms |
| `jwtFormat` | `"simple" \| "aap"` | `"simple"` | JWT claim format |
| `jwtMaxAge` | `number` | `60` | Maximum JWT age in seconds |
| `agentSessionTTL` | `number` | `3600` | Sliding session TTL in seconds (0 = no expiry) |
| `agentMaxLifetime` | `number` | `86400` | Hard max agent session lifetime in seconds |
| `maxAgentsPerUser` | `number` | `25` | Max active agents per user (0 = unlimited) |
| `validateScopes` | `boolean \| function` | `undefined` | Validate scopes on agent creation |

## Schema

### Agent

| Field | Type | Description |
|---|---|---|
| `id` | string | Unique agent identifier (primary key) |
| `name` | string | Friendly name |
| `userId` | string | Owner user ID (foreign key) |
| `orgId` | string? | Organization ID |
| `workgroupId` | string? | Workgroup ID (foreign key) |
| `scopes` | string | JSON array of scope strings |
| `role` | string? | Role name |
| `status` | string | `"active"` or `"revoked"` |
| `publicKey` | string | Ed25519 public key as JWK JSON |
| `kid` | string? | Key ID from the JWK |
| `lastUsedAt` | date? | Last authenticated request |
| `metadata` | string? | JSON metadata |
| `createdAt` | date | Creation timestamp |
| `updatedAt` | date | Last modification timestamp |

### Agent Scope Request

| Field | Type | Description |
|---|---|---|
| `id` | string | Primary key |
| `agentId` | string | Agent requesting escalation |
| `userId` | string | Owner user ID |
| `agentName` | string | Agent name at time of request |
| `newName` | string? | Requested name change |
| `existingScopes` | string | JSON array of current scopes |
| `requestedScopes` | string | JSON array of requested scopes |
| `status` | string | `"pending"`, `"approved"`, or `"denied"` |
| `createdAt` | date | Request timestamp |
| `expiresAt` | date | Request expiration |

### Agent Workgroup

| Field | Type | Description |
|---|---|---|
| `id` | string | Primary key |
| `name` | string | Workgroup name |
| `description` | string? | Description |
| `orgId` | string? | Organization ID |
| `createdAt` | date | Creation timestamp |
| `updatedAt` | date | Last modification timestamp |
